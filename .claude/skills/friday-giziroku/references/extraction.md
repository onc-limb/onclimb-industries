# 構造抽出の判定基準（Phase D）

口語の文字起こしから「会話の流れ / 決定 / 保留 / TODO」を切り分ける基準。
**決定とアイデアの混同**を避けることが品質の肝。

## 会話の流れ

- 時系列だが、発言単位ではなく **議題単位**でダイジェストにする。
- 同じ論点の往復は 1 つにまとめる。言い直し・相槌・フィラー（「えーと」「なんか」）は落とす。
- 脱線・雑談は復元せず「(雑談: 〜)」と 1 行に圧縮する。
- タイムスタンプがあれば議題の頭に目安として残してよい（必須ではない）。

## 全体要約

- 3〜5 行。読み手が本文を読まずに要点を掴める粒度。
- 「何を話し / 何が決まり / 次に何をするか」を含める。

## 決定事項 vs 保留（最重要の切り分け）

| 決定事項に入れる | 保留・継続検討に回す |
|---|---|
| 「〜することにします / 決めました / でいきましょう」と**明示的に合意**したもの | 「〜かな」「〜した方がいいかも」「検討したい」の言いさし・仮定法 |
| 担当・期日まで握れたもの | 方針は出たが判断材料待ちのもの |
| 全員（または決定権者）が同意したと読めるもの | 一人が提案しただけで合意が読めないもの |

- 迷ったら**決定に入れず保留**に置く（誤って「決定」とすると手戻りが大きい）。
- 各決定に `[D1] [D2]…`、各保留に `[P1] [P2]…` の通し番号を振る。
- 決定者が読み取れれば `（決定者: 氏名）` を添える。

## TODO / アクションアイテム

- 「〜やっておきます」「〜直します」「〜連絡します」など**行動の引き受け**を拾う。
- 必ず **担当**と**期日**を付ける。読み取れない場合も空欄にせず明示する:
  - 担当不明 → `担当: 不明`
  - 期日不明 → `期日: 未定`
- 期日表現は原文を尊重しつつ補う:「本日中」「水曜（ドラフト）」「次の週」など。
- 決定事項と重複してよい（決定の実行タスクは TODO にも書く）。
- テンプレの TODO テーブルでは、決定事項に由来するタスクは**関連列に対応する決定番号 `[Dn]`** を
  記して紐付ける（決定由来でなければ `−`）。

## 完了判定（空欄で終わらせない）

- 決定事項・TODO が 0 件のとき → 「抽出できなかった」ではなく
  **「雑談主体で決定/TODO なし」**などと判断根拠を本文に明記する。
- 1 件も話者が確定できない／文字起こしが壊れて読めない場合は、生成を強行せず
  Phase B に戻ってユーザーに確認する。

## 補足セクション

- **誤変換の補正メモ**: Phase C で直した固有語・専門語を `元 → 補正` で列挙（推測痕跡）。
- **不明点・要確認**: 文脈不足で推測した箇所、聞き取れず曖昧な箇所を残す。読み手の検算用。

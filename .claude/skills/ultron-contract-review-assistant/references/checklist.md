# 業務委託契約書 要注意条項チェックリスト

フリーランスエンジニア（**受託者**の立場）向け。SKILL.md の標準フロー手順 3 で、
**すべての観点を上から順に**契約書と照合し、各観点について
「該当条項あり（原文引用＋条番号）/ 記載なし」を必ず判定する。

- 各観点の「確認文例」は、そのまま先方に送れる文面のたたき台。契約書の実際の条番号・文言に合わせて調整する。
- 法令への言及は本ファイル最終更新時点の一般的な理解。最新の法令・運用は一次情報（e-Gov、公正取引委員会、厚生労働省）で確認する。
- 見落としが見つかったら本ファイルに観点・パターンを追記する。

## 1. 契約類型（準委任か請負か）と業務範囲

**なぜ重要か**: 類型によって、成果物の完成義務の有無・報酬請求の条件・契約不適合責任の有無が変わる。
タイトルが「業務委託契約」でも、法的には準委任（民法 656 条・643 条）か請負（民法 632 条）のどちらか（または混合）。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 業務内容・委託内容の条項 | 「成果物を完成させ納入する」＝請負寄り。「業務の遂行」「善良な管理者の注意をもって」＝準委任寄り |
| 検収・報酬の条項 | 準委任のはずなのに「検収合格をもって報酬を支払う」→ 実質請負（未完成リスクを負う）の可能性 |
| 業務範囲の定義 | 範囲が「関連する一切の業務」等と曖昧 → 無償の追加作業を求められる余地 |
| 別紙・個別契約への委任 | 業務内容が別紙・注文書任せで本紙に無い → 別紙も必ず入手して確認 |

- **確認文例**: 「本契約は準委任（成果物の完成義務を負わない）と請負のどちらの性質を想定されていますか。第◯条の検収の定めとの関係を確認させてください。」
- **確認文例（範囲）**: 「第◯条の業務範囲について、含まれない作業（例: 運用保守・障害対応・ドキュメント作成）を別紙で明確化させていただけますか。」

## 2. 知的財産権の帰属

**なぜ重要か**: 書き方次第で、過去に作った汎用ライブラリや、業務で得たノウハウの再利用までできなくなる。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 著作権譲渡の条項 | 「著作権法第 27 条及び第 28 条の権利を含め譲渡」＝翻案権まで全部譲渡（特掲がなければ 27/28 条は譲渡されないのが原則・著作権法 61 条 2 項） |
| 譲渡対象の範囲 | 「本件業務に関連して生じた一切の知的財産権」→ 既存資産・汎用モジュール・ノウハウまで巻き込む書き方 |
| 著作者人格権 | 「著作者人格権を行使しない」不行使特約は一般的だが、範囲が業務成果物に限定されているか |
| 発明・ノウハウ | 職務発明的な条項が個人事業主に適用される書き方になっていないか |
| 実績公開 | 成果物・案件名をポートフォリオ等で公開してよいかの定めの有無 |

- **受託者に有利な形**: 譲渡対象を「本件業務により新たに作成した成果物」に限定し、既存著作物・汎用モジュールは**利用許諾**にとどめる。
- **確認文例**: 「第◯条の著作権譲渡について、本件業務の開始前から当方が保有する既存プログラム・汎用モジュールは譲渡対象外（貴社には利用許諾）という理解でよいでしょうか。」

## 3. 競業避止・専念義務

**なぜ重要か**: フリーランスは複数案件の並行が前提。広い競業避止は事実上の専属化＝収入源の制限になる。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 競業避止条項 | 「同業他社の業務を行ってはならない」— 期間・地域・対象業務が無限定 |
| 契約終了後の効力 | 「契約終了後◯年間」— 終了後まで及ぶ場合、期間と対価の有無 |
| 専念・専属条項 | 「委託業務に専念する」「他の業務委託を受ける場合は事前承諾」→ 並行案件の可否 |
| 引き抜き禁止 | 顧客・従業員への勧誘禁止の範囲が広すぎないか |

- **確認文例**: 「第◯条の競業避止について、当方は複数社と業務委託契約を締結しております。制限の対象は『本件業務で知り得た機密を利用した競業』に限定していただくことは可能でしょうか。」

## 4. 損害賠償（上限・範囲）

**なぜ重要か**: 個人が無限定の賠償責任を負うと、1 件の事故で事業が破綻し得る。**上限の定めが無いこと自体が最大のリスク**。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 損害賠償条項 | 上限の定めが無い / 「一切の損害を賠償する」 |
| 賠償範囲 | 「間接損害・逸失利益を含む」→ 範囲が広い。直接かつ現実に生じた通常損害への限定があるか |
| 上限額の相場 | 「委託料の◯か月分」「本個別契約の委託料相当額」を上限とする定めが受託者側の一般的な落とし所 |
| 片務性 | 受託者のみが賠償義務を負い、委託者側の義務が無い片務的な書き方 |
| 免責の例外 | 故意・重過失の場合は上限が外れる建付けが一般的（それ自体は通常想定内） |

- **確認文例**: 「第◯条の損害賠償について、賠償額の上限を本件業務の委託料◯か月分相当額とし、対象を直接かつ現実に生じた通常損害に限定する修正はご検討いただけますか。」

## 5. 報酬・支払サイト

**なぜ重要か**: キャッシュフローに直結。フリーランス保護新法により、報酬支払期日は**給付受領日から 60 日以内**が原則（§9 参照）。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 報酬額・算定方法 | 金額・時間単価・精算幅（例: 140–180h）と超過/控除単価が明記されているか |
| 支払期日 | 「月末締め翌々月末払い」等 → 受領から 60 日超は新法上問題の可能性。締め日と支払日を具体的に確認 |
| 支払条件 | 「検収合格後に支払う」「委託者がエンドユーザーから入金を受けた後に支払う」（入金連動）→ 支払が不当に遅延・不確定になる |
| 諸経費 | 交通費・機材・ライセンス費の負担者が明記されているか |
| 消費税 | 税込/税抜の明記、インボイス（適格請求書）発行の要否 |
| 減額の余地 | 「業務の状況により報酬を減額できる」等の一方的減額条項（新法の禁止行為に当たり得る） |

- **確認文例**: 「報酬の支払期日について、フリーランス取引適正化法では業務提供を受けた日から 60 日以内の支払が求められていると理解しています。第◯条の支払サイトはこの範囲に収まる運用でしょうか。」

## 6. 中途解約・契約期間

**なぜ重要か**: 突然の契約打ち切りは翌月の収入がゼロになるリスク。予告期間と既履行分の精算が生命線。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 中途解約条項 | 委託者が「いつでも即時に解約できる」一方、受託者側の解約に制限がある片務的な定め |
| 予告期間 | 予告期間の定めが無い / 極端に短い。（新法では 6 か月以上の継続的業務委託の中途解除は原則 30 日前までの予告が必要・§9） |
| 既履行分の精算 | 解約時に「実施済みの業務に対する報酬」を日割り等で支払う定めがあるか |
| 契約期間・更新 | 自動更新の有無、更新拒絶の通知期限（更新しない場合◯日前通知） |
| 解約時の損害 | 受託者からの解約に違約金・損害賠償が課される建付けになっていないか |

- **確認文例**: 「第◯条の中途解約について、解約予告期間（例: 30 日前）と、解約日までに実施した業務分の報酬の精算方法を明記していただくことは可能でしょうか。」

## 7. 瑕疵担保 / 契約不適合責任

**なぜ重要か**: 請負（または成果完成型準委任）では納品後の修補・賠償責任を負う。期間・範囲が無限定だと納品後もリスクが続く。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 契約不適合責任の条項 | 責任期間の定めが無い（民法の原則では不適合を知った時から 1 年以内の通知）。「検収後◯か月（例: 3〜6 か月）」等の限定があるか |
| 「瑕疵担保」の残存 | 2020 年民法改正前の「瑕疵担保責任」の用語のまま → 現行法との対応関係を要確認 |
| 責任の内容 | 無償修補に加え、損害賠償・報酬減額・解除まで広く認める書き方か。§4 の賠償上限と連動しているか |
| 準委任なのに規定 | 準委任（履行割合型）なのに成果物の契約不適合責任を負わせる条項 → 類型と不整合（§1 と併せて確認） |
| 仕様変更起因の除外 | 委託者の指示・仕様変更に起因する不具合が責任対象から除外されているか |

- **確認文例**: 「第◯条の契約不適合責任について、責任期間を検収完了後◯か月以内に限定し、対象を当方の責めに帰すべき事由による不適合に限る形にしていただくことは可能でしょうか。」

## 8. 再委託

**なぜ重要か**: 全面禁止だと繁忙時に外注できない。逆に自分が中間に立つ場合は再委託先の責任を全部負う建付けに注意。

| 見るべき箇所 | 要注意パターン |
|---|---|
| 再委託条項 | 全面禁止か、事前の書面承諾で可か。（一人で完結する SES 常駐案件なら全面禁止でも実害は小さい） |
| 再委託時の責任 | 再委託先の行為について受託者が「一切の責任を負う」→ §4 の賠償上限が及ぶか確認 |
| 秘密保持との連動 | 再委託先にも同等の秘密保持義務を課す旨（一般的で妥当） |

- **確認文例**: 「第◯条の再委託について、事前に書面で承諾いただいた場合には一部業務の再委託が可能、という形にしていただくことは可能でしょうか。」

## 9. フリーランス保護新法（特定受託事業者に係る取引の適正化等に関する法律）の観点

**なぜ重要か**: 2024 年 11 月施行。従業員を使用しないフリーランス（特定受託事業者）への業務委託に適用され、
下請法より広く**資本金要件なし**で発注者に義務を課す。契約書がこの義務と整合しているかを確認する。

| 確認項目 | 内容・要注意パターン |
|---|---|
| 取引条件の明示（3 条） | 業務内容・報酬額・支払期日等を**書面または電磁的方法で明示**する義務。口頭発注前提・条件が別紙未提出のままは要注意 |
| 報酬支払期日（4 条） | **給付を受領した日から 60 日以内**のできる限り短い期間内に支払期日を設定。「翌々月末払い」等は締め日次第で 60 日超の可能性 |
| 禁止行為（5 条） | 1 か月以上の継続的業務委託で: 受領拒否 / 報酬の減額 / 返品 / 買いたたき / 物品・役務の購入・利用強制 / 不当な経済上の利益の提供要請 / **不当な給付内容の変更・やり直し**。契約書に「委託者は無償で仕様変更・やり直しを求められる」趣旨の条項があれば要注意 |
| 募集情報の的確表示（12 条） | 募集時の条件と実際の契約条件が食い違っていないか |
| 育児介護等への配慮（13 条） | 6 か月以上の継続的業務委託では、育児・介護等との両立への配慮義務 |
| ハラスメント対策（14 条） | 発注者側にハラスメント対応の体制整備義務 |
| 中途解除等の予告（16 条） | **6 か月以上**の継続的業務委託の解除・不更新は、原則 **30 日前までに予告**。契約書の解約条項（§6）がこれより短い予告で解約できる建付けなら要注意 |

- 適用対象・期間要件（1 か月 / 6 か月）は取引の形態によるため、当てはめは断定せず「新法の適用可能性」として提示する。
- **確認文例**: 「本契約はフリーランス・事業者間取引適正化等法の対象になると理解しています。取引条件の明示（業務内容・報酬・支払期日）と、継続的業務委託の場合の中途解除 30 日前予告について、契約書上の取り扱いを確認させてください。」

## 10. その他の一般条項

| 観点 | 要注意パターン | 確認文例の方向性 |
|---|---|---|
| 秘密保持 | 秘密情報の定義が無限定 / 有効期間が無期限 / 受託者のみ義務を負う片務型 | 対象の定義と期間（例: 終了後 3 年）の明確化 |
| 指揮命令（偽装請負リスク） | 業務委託なのに委託者が勤務時間・場所・手順を直接指示する前提の条項（SES で特に注意） | 指示系統・進捗報告の方法を「協議」「依頼」ベースに |
| 反社会的勢力の排除 | 条項が無い（通常は双方に入れる。無いこと自体が契約書の完成度のシグナル） | 反社条項の追加提案 |
| 合意管轄 | 遠方の裁判所の専属管轄（紛争時の負担増） | 双方の折衷（例: 被告所在地）や自分の所在地への変更打診 |
| 権利義務の譲渡禁止 | 一般的な条項。報酬債権の譲渡（ファクタリング利用）まで縛られるかを必要に応じ確認 | — |
| 存続条項 | 契約終了後も存続する条項（秘密保持・競業避止・賠償）の範囲が広すぎないか | 存続対象と期間の明確化 |
| 契約の変更 | 「委託者は通知により条件を変更できる」等の一方的変更条項 | 双方の書面合意による変更へ修正打診 |

# 配当書類の読み取りガイド

配当金計算書・配当金支払通知書のレイアウトは信託銀行（証券代行）ごとに異なるが、
記載項目はほぼ共通。DividendRecord への対応付けを固定する。

## 項目対応表

| 書類上の表記（ゆれ含む） | フィールド | 備考 |
|---|---|---|
| 銘柄名 / 会社名（「株式会社◯◯」等の発行会社名） | `stockName` | 書類の発行元（信託銀行・証券代行）と混同しない。証券コードが読めれば「銘柄名(コード)」形式も可 |
| 1株あたり配当金 / 1株当たり配当金 | `dividendPerShare` | 円。小数あり（例: 19.00） |
| 所有株数 / 持株数 / ご所有株式数 | `shares` | 株 |
| 配当金額 / 配当金 | `amount` | **税引前**。これを記録する |
| 支払金額 / お支払金額 | （記録しない） | 税引後の手取り。`amount` と混同しない。NISA・非課税なら配当金額と同額になる |
| 基準日 / 割当基準日 | `recordDate` | YYYY-MM-DD に正規化 |
| 支払確定日 / 支払開始日 | `recordDate` の代用 | **基準日の記載が無い場合のみ**代用し、その旨をユーザーに伝える |

## 読み取り時の注意

- **配当金額 vs 支払金額**: 税額欄が `*` 埋め（非課税・NISA 等）の書類は両者が同額。
  課税ありの書類は 支払金額 = 配当金額 − 税額 となるため、必ず「配当金額」欄を採る。
- **検算**: 一株配当 × 株数 = 配当金額 が成り立つはず（端数処理で ±1 円まで許容）。
  ズレたらどこかを誤読している可能性が高いので、確認時に必ず両方の値を見せる。
- **期の表記**: 「第◯期（YYYY年M月D日〜YYYY年M月D日）」は事業年度であって基準日ではない。
  基準日が無い場合の代用は支払確定日を優先する。
- **読めない値**: 数値は 0、文字列は空文字にして「要確認」と明示する（もっともらしい値を推測で埋めない）。
- **中間/期末**: 同一銘柄でも中間配当と期末配当は基準日が異なる別レコード。重複ではない。

## Notion 過去データの移行マッピング

Notion の手打ち記録を CSV 化するときの列対応。Notion 側の列名は運用によってゆれるため、
実際のエクスポートを見て下表に当てはめ、対応関係をユーザーに確認してから import する。

| Notion 側の想定列 | CSV 列（import-csv の入力） | 必須 | 欠損時 |
|---|---|---|---|
| 銘柄名 / 銘柄 / 会社名 | `stockName` | ✔ | — |
| 基準日 / 受取日 / 日付 | `recordDate`（YYYY-MM-DD） | ✔ | — |
| 配当金額 / 金額 | `amount` | ✔ | — |
| 1株配当 | `dividendPerShare` | | 0 |
| 株数 | `shares` | | 0 |
| （なし） | `sourceImage` | | `notion-migration` が自動で入る |

- 日付が「2026年6月2日」「2026/6/2」形式なら YYYY-MM-DD に正規化する。
- 金額の「¥」「,」「円」は取り除いて数値のみにする。
- 同一銘柄・同一日・同一金額の行が Notion 側に本当に 2 件ある場合（同日に複数口座で受領等）は、
  ドライランのスキップ一覧で検出されるので、ユーザー確認のうえ `--force` で個別に入れる。

# people-memory — 人物メモリ（会話の記録と好みメモ）

雑談・家族との会話の文字起こしを入力に、**人物ごとのノートに会話記録を追記して育てる**。
生成するのは 1 回ごとの文書ではなく、**1 人 = 1 ノートの台帳**。
persona: [`personas/vision.md`](../../../personas/vision.md)（読み手は自分だけ・評価を書かない・推測は明示）。

> **起動条件（重要）**: ユーザーが会話ログ・文字起こしを指して人物メモリへの記録を
> 明示的に依頼したときだけ走る。自動起動しない。
> **決定事項・TODO を残すことが主目的**の録音は [friday-giziroku](../friday-giziroku/SKILL.md)
> （`meeting` / `informal_decision` テンプレ）の領分。同じ文字起こしに「決定」と
> 「人となりのネタ」が両方あるときは、giziroku で議事録化した後に本スキルへも食わせてよい
> （その場合の入力退避は先に処理したスキル側の processed/ を参照し、二重移動しない）。

## 場所（コードとデータは分離）

- ツール・テンプレ: このスキルディレクトリ `.claude/skills/vision-people-memory/`
- **入力（文字起こし）**: リポジトリ直下 `people-memory/transcripts/`。ユーザーが他の場所を
  指定した場合はそのパスを使う。
- **人物ノート（正本）**: `people-memory/people/<slug>.md`。**1 人 = 1 ノート**。
  slug は英小文字ケバブケース（例: `tanaka-taro`, `wife`）。本名を出したくない相手は
  呼び名 slug でよい（ノート内の表記もその呼び名に揃える）。
- **処理済みの入力**: 追記に成功したら `people-memory/processed/` へ**移動して退避**
  （削除しない＝原文を残す）。
- `people-memory/` は人間関係・家族の機密を含むため **git 管理外**（`.gitignore` 済み）。
  内容を外部サービスへ送信しない。ただし**ユーザー自身の Google Drive
  （マイドライブ/onclimb-industries/会話ログ/）への同期のみ許可**（2026-07-14 ユーザー指示。
  transcripts / processed / people は同フォルダへのシンボリックリンクになっている）。
- 下処理資産は friday-giziroku と**共有**する:
  - 解析スクリプト: `.claude/skills/friday-giziroku/scripts/parse_transcript.py`
  - 誤変換辞書・名簿（実データ・git 管理外）: `giziroku/config/glossary.yaml` / `giziroku/config/roster.yaml`
  - 本スキルで確定した話者マッピング・誤変換訂正も **giziroku/config/ 側に追記**する
    （両スキルで使うほど補正と話者推定が当たるようになる）。

## トリガーと非トリガー

| 起動する発話の例 | 起動しない（誤起動を避ける）|
|---|---|
| 「この雑談、〇〇さんのメモに追記して」 | 「議事録にして」「決まったこと拾って」→ friday-giziroku |

## 4 段フロー（giziroku と同じ骨格。順序は固定）

「**対話で前提を固める → 機械的に下処理 → 抽出 → 人物ノートへ追記**」。

### Phase A. 一次パース（機械処理）

1. 対象ファイルパスを受け取る（未指定なら `people-memory/transcripts/` の中身を `ls` で提示し選ばせる）。
2. giziroku の解析スクリプトで素材（話者集計・抜粋・タイムライン・分離有無）を得る:
   ```bash
   python3 .claude/skills/friday-giziroku/scripts/parse_transcript.py "<transcript_path>" --json
   ```
3. `people-memory/people/` の既存ノート一覧と `giziroku/config/roster.yaml` から、
   話者候補の当たりを付ける（確定はしない）。

### Phase B. 対話で前提確定 ★

`AskUserQuestion` で **3 点に絞って**確認する。推定値を初期選択にする。

1. **話者マッピングと記録対象**: 話者ごとに発言量＋抜粋を提示し「誰か」を確定する。
   選択肢は既存ノートの人物 ＋ roster 候補 ＋ 自由入力。**本人（ユーザー自身）以外の
   参加者それぞれが記録対象**（複数人の雑談なら人数分のノートに追記する）。
   記録しなくてよい人物は「対象外」を許可する。
2. **関係種別**: `work`（同僚・仕事関係）/ `friend`（友人・知人）/ `family`（家族）。
   新規人物のときだけ聞く（既存ノートは frontmatter から引き継ぐ）。
   種別で抽出フォーカスを切り替える（[`references/extraction.md`](references/extraction.md)）。
3. **会話日**: ファイル名・先頭メタ・タイムスタンプから推定して確認。採れないときだけ聞く
   （当日日付で埋めない）。

### Phase C. 下処理（ルール確定のみ）

- `giziroku/config/glossary.yaml` と文脈で固有語・誤変換の補正ルールを確定する。
  補正はノートに書くエントリの文面にのみ適用し、原文は無改変のまま退避する。
- **マスキングはしない**（読み手は自分だけ・データは git 管理外）。ただし API キー等の
  機密文字列が紛れていたら記録に含めない。

### Phase D. 抽出と人物ノートへの追記

1. [`references/extraction.md`](references/extraction.md) の基準で、記録対象の人物ごとに抽出する:
   - 話した話題（見出しになるキーワード）
   - 分かった事実・近況（本人の発言に基づくもの）
   - 好み（好き / 苦手 / 欲しがっていた物・気になっていた物）— **family は厚めに**
   - 大事な日付（記念日・予定・締切）
   - 次に会うとき・気にかけること（持ち越した話題、約束）
   - 事実と推測を分け、推測には「(推測)」＋根拠の発言を添える。
2. 人物ノートが無ければ [`templates/person_note.md`](templates/person_note.md) から新規作成する。
   既存ノートには:
   - 「会話ログ」セクションの**先頭**に今回のエントリ（[`templates/entry.md`](templates/entry.md)）を追記（新しい順）。
   - 「プロフィールメモ」「好み・嗜好」「大事な日付」に恒常的な情報を反映する。
     既存の記述と矛盾したら黙って上書きせず、`(更新: YYYY-MM-DD 旧「〜」→ 新「〜」)` と履歴を残す。
   - 過去の会話ログエントリは書き換えない（訂正は新エントリで）。
3. **入力の退避**: 追記に成功したら入力を退避する（削除しない）:
   ```bash
   mkdir -p people-memory/processed && mv "<transcript_path>" people-memory/processed/
   ```
   `transcripts/` 以外を直接指定された入力は、退避してよいか一度確認する。
   同名ファイルが既にあれば上書きせず連番/日付を付ける。
4. 出力後、チャットに要約（追記した人物・拾えた話題数・好みの更新有無・退避先）を返す。
5. **ToDo 台帳への収穫**: 会話から「あとでやる」が拾えたら（例: プレゼントを買う、
   〇〇を送ると約束した）、候補として提示し確認のうえ
   `todo.py add --status inbox --source-type session --source-ref <ノートパス>` で台帳に入れる
   （追記のたびに一言通知）。

### 参照モード（記録ではなく読むだけ）

「〇〇さんと最近何話したっけ」「〇〇の好みなんだっけ」には、該当ノートを読んで
チャットで答えるだけ。ノートの変更・入力の退避はしない。該当ノートが無ければ無いと答える。

## 自己進化（実行ごとに資産が育つ）

固定する芯は崩さない（起動条件・4 段フロー・事実と推測の分離・評価を書かない）。蓄積するのは資産:

- 人物ノートそのものが主資産（会話のたび厚くなり、参照の精度が上がる）。
- 確定した話者マッピング → `giziroku/config/roster.yaml` に追記（giziroku と横断共有）。
- 訂正された誤変換 → `giziroku/config/glossary.yaml` に追記（同上）。

## やってはいけないこと

- 自動起動しない。明示依頼が無ければ走らない。
- 人物の評価・値踏み・格付けを書かない（話題・事実・発言の記録に徹する）。
- 推測を事実として書かない（「(推測)」＋根拠なしの好み記載は禁止）。
- 過去エントリを書き換えない（訂正は新エントリ・更新履歴で）。
- 原文を削除しない（processed/ への退避のみ）。
- `people-memory/` の内容を外部サービスへ送信しない。git 管理下に置かない。
  例外はユーザー自身の Google Drive への同期のみ（上記「置き場所」参照）。
- 決定事項・TODO 中心の録音を本スキルで処理しない（giziroku へ案内する）。

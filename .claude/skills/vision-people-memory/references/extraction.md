# 抽出基準（人物ノートに何を拾い、何を捨てるか）

議事録（giziroku）の「決定・保留・TODO」とは基準が違う。ここで拾うのは
**その人がどういう人か・何を話したか**であって、仕事の結論ではない。

## 共通の抽出対象（全関係種別）

| 区分 | 拾うもの | 例 |
|---|---|---|
| 話題 | 会話の見出しになるキーワードと要点 | 「引越しの話」「最近見たドラマ」 |
| 事実・近況 | 本人の発言に基づく出来事・状況の変化 | 「来月から部署異動」「子どもが入学した」 |
| 好み | 好き / 苦手 / 欲しがっていた・気になっていた物事 | 「辛いものが苦手」「新しいキーボードを見ていた」 |
| 大事な日付 | 誕生日・記念日・本人にとって大きい予定 | 「結婚記念日が 8 月」「資格試験が 9 月」 |
| 約束・持ち越し | 自分がその人と交わした約束、次に続く話題 | 「本を貸す」「次はあの店に行こうと言った」 |

## 事実と推測の線引き（最重要）

- **事実**: 本人が明言したこと。「〜が好きって言ってた」レベルはそのまま書いてよい。
- **推測**: 発言や反応からの読み。必ず `(推測)` を付け、根拠の発言を引用で添える。
  - 例: `(推測) キャンプに興味がありそう — 根拠: 「〇〇のギア動画ばかり見てる」`
- 根拠を添えられない印象論（「たぶん〜なタイプ」）は書かない。
- 人物の評価・値踏み（優秀/微妙、好き/嫌い等の採点）は事実でも推測でも書かない。

## 関係種別ごとのフォーカス

### work（同僚・仕事関係）
- 話題・近況・持ち越しを中心に。次に会うときの会話の取っ掛かりを残すのが目的。
- 仕事の決定・TODO が主役になっていたら、それは giziroku（informal_decision）の領分。
  人となりのネタだけをこちらに拾う。

### friend（友人・知人）
- work と同じ構成で、趣味・近況・約束を厚めに。

### family（家族）
- **好み・嗜好を最優先で厚めに拾う**（プレゼント・献立・出かけ先の選定に効く）:
  - 明示の好み（「これ好き」「あれは苦手」）
  - 欲しがっていた・買おうか迷っていた物
  - 喜んだ / 気乗りしなかった反応（推測タグ付きで）
- 大事な日付（記念日・学校行事・通院など）と約束は取りこぼさない。
- 体調・悩みなどセンシティブな話題は、本人が明言した事実の範囲でだけ短く残す
  （憶測で膨らませない）。

## 捨てるもの

- 相槌・つなぎ・その場限りの時事ネタ（後から引く価値がないもの）。
- 会話に登場しただけの第三者の詳細（記録対象はあくまで会話の相手）。
  ただし相手との関係で大事な情報（例: 家族の話）は相手のノートに残してよい。
- API キー等の機密文字列、業務上の秘密そのもの。

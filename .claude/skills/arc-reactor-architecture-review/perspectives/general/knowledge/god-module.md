# God モジュール・ファンイン/ファンアウト過多

- タグ: smell, coupling
- シグナル: 参照される数（ファンイン）/ 依存する数（ファンアウト）が、リポジトリ内の
  同種モジュールと比べて突出している。1 つのモジュール/クラスに複数責務が集まっている。
  内部実装（private・具体型・データ構造）に多数の箇所が依存している。
- なぜ問題か: あらゆる変更がそこを通るため、変更の衝突・退行リスクが一点に集中する。
  理解にもそのモジュール全部の読解が必要になる。
- 検出方法: `git grep` で import 元を数えて相対比較する。目安は**相対評価** —
  同種モジュールの中央値の 3 倍超、または最大級（初期値）。絶対数の閾値は使わない（規模依存のため）。
- 重要度の目安: 通常は imo（責務の切り出し提案付き）。変更が常にそこを通りテスト不能に
  なっている等の実害が確認できれば must。
- 例外: 意図された共通基盤（logger・設定・型定義集約など）はファンインが高くて正常。
  問題は「参照が多い」ことではなく「責務が混ざっている」こと。

# 循環依存

- タグ: coupling
- シグナル: モジュール A↔B が相互に import/参照している。変更が常に両側へ波及する。
- なぜ問題か: 片方だけを理解・変更・テストできなくなる。初期化順やビルドの実害も出る。
- 検出方法: `git grep -nE "^\s*(import|from|require|use )" -- <対象パス>` で import を拾い
  相互参照を探す。対象リポジトリに導入済みのツールがあれば活用
  （JS: `madge --circular`、Go: `go mod graph`、Python: `pydeps`/`import-linter`）。
  新規インストールはしない（`npx`/`uvx` の一時実行のみ可）。
- 重要度の目安: 初期化順・ビルドに実害がある、または変更が常に両側へ波及する密結合なら must。
  型の共有だけの緩い循環など実害が読み切れないものは imo（解消案付き）か ask。
  解消案はまず「共有する型の別モジュールへの切り出し」か「片方向への整理」。
- 例外: 意図された相互再帰（AST ノード群など、ドメイン自体が再帰的なもの）。

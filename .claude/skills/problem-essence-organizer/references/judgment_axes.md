## 軸 1. 課題 (Problem) と 手段 (Solution) の見分け方

### 課題 (Problem) の特徴

- **誰が** 何に困っているかが具体的 (主語が「ユーザー / 顧客 / 業務」)
- **状態の差分** で語られる (「X ができない」「Y に N 時間かかる」「Z で失敗が頻発」)
- **手段に依存しない** (どんな手段でも成立する困りごと)
- **観測可能** (現状の数値・体験で示せる)

### 手段 (Solution) の特徴

- 主語が **道具・技術** ("RAG が"、"Agentic Search が"、"LLM が")
- **どう実装するか** の話 (アーキ・スタック・モデル選定)
- **困りごとと因果関係が明示されていない**
- 「これを使って」「これで作って」と語られる

### 判定フロー (round 5 で確立)

```
発話を読む
  ├─ 主語が「人 / 業務 / 状態」 → 課題寄り
  ├─ 主語が「技術 / ツール」    → 手段寄り
  └─ どちらも含む               → 「なんのためにそれを使うのか?」で分離
```

### 典型的な「手段先行」パターン

- 「RAG を使ったチャットボットを作りたい」 → 手段先行。F1 へ戻す。
- 「Agent 検索の精度を上げたい」 → 単体では Done 基準なし。F3 を要確認。
- 「LLM のレスポンス速度を改善したい」 → 何のための速度? を遡る。

## 軸 2. Done 基準が定義されているかの見分け方

### Done と呼べる条件

- **観測可能** (誰が見ても判断できる)
- **時間・量・状態の閾値** が含まれる ("〜以内に"、"〜以上の"、"〜の状態で")
- 主語が **誰 (User / Customer / Business)** にとってかが明示
- 手段ではなく **結果の状態** で書かれている

### Done と呼べないもの

- 「精度を上げる」「速度を上げる」 → 何 % / 何秒、誰にとって、を要追加
- 「ユーザーの満足度を上げる」 → どう測るか不明
- 「動くものを作る」 → 状態ではなく行為

### F3 違反の検知 (重要)

F4 (手段検討) に入ろうとしているのに、上記の Done 条件を満たしていない → **F3 へ戻す**。
このとき返す質問は `references/phase_questions.md` の F3 セクションから選ぶ。

## 軸 3. 中心 (重要) と 周辺 (付随) の見分け方

ユーザーの本意 (round 7 で確定): **二項分離 (注目 vs 脇に置く)** の 1 軸。
ラベル名 ("コア/外枠"、"課題/派生手段") はバリエーション。本質は中心 vs 周辺。

### 中心 (重要 / 注目) に置くもの

- 課題そのもの (誰の何の困りごとか)
- Done 基準 (何が達成されたら解決か)
- 制約条件 (時間・予算・関係者・依存)
- 不足情報 (今、何が決められないか)

### 周辺 (付随 / 脇に置く) に置くもの

- 具体的な技術手段 (RAG / Agentic / DB 選定など、Done が決まる前の段階では基本「脇」)
- 思いつきレベルのアイディア
- 過去にやって失敗した方法 (記録として保持するが目立たせない)
- 自分の得意領域に引き寄せた解釈

### 適用ルール

- md 出力では、各章を「重要」「付随」の 2 セクションに必ず分離する。
- 注意が向きやすいもの (= 手段) ほど **意識的に「付随」側に書く** ことで引きずられない。

## 軸 4. 現在地のフェーズ判定 (F1〜F4)

### F1 (課題発見) の特徴

- 課題の輪郭が言語化されていない
- 主語が技術になっている
- 「やりたいこと」だけがある (なぜ・誰のため不明)

### F2 (課題整理) の特徴

- 課題候補が複数列挙されている
- 重なりや包含関係が未整理
- 優先度がついていない

### F3 (解決定義) の特徴

- 中心課題は決まっている
- だが Done 基準が抽象 / 測定不能 / 不在

### F4 (手段検討) の特徴

- 課題と Done が両方明確
- 手段の候補比較に入っている (or 入ろうとしている)

### 戻りトリガー (逆流ルール)

| 現在地 | 戻すトリガー | 戻す先 |
|---|---|---|
| F4 | Done 不明 / 「目的ってなんだっけ」発話 | F3 |
| F4 | 課題そのものが不明 / 課題が手段になっている | F1 |
| F3 | 課題が複数で絞れていない | F2 |
| F2 | どの候補も手段で記述されている | F1 |
| any | ユーザーが「落ち着きたい」「沼にハマってる」発話 | F3 (Done 確認) → 必要なら F1 へ |

# フェーズ別 定型問いセット (進化対象)

各フェーズで投げるオープン質問・キラーフレーズの台帳。`references/dialogue_style.md` 3 原則に準拠。
進化対象。使用ログで「ハッとした」反応の多い問いを優先度上げ、効かなかった問いを差し替える。

---

## キラーフレーズ (短い割り込み・モード a 用)

最大 1 行。会話に挟む。

- 「何を達成したらゴール?」
- 「なんのためにやるの?」
- 「(手段名) じゃなくてもいいのか?」
- 「今、誰のために何を解こうとしてる?」
- 「それで本当にいいのか?」
- 「目的とずれてない?」

---

## F1: 課題発見 (Discovery)

### 起動条件
- 課題の形が決まっていない
- 手段ベースの要望が出ている (「○○を使って○○を作りたい」)
- クライアントが商談の最初期にいる

### 投げる問い

1. **誰を対象に?** (Who)
2. **何を叶えるため?** (Why / For what)
3. **その手段じゃなくてもいいのか?** (Solution-agnostic check)
4. その人/業務は今、具体的に何に困っている? (Current pain)
5. 困っている状態と困っていない状態の差は、どう観測できる?

### F1 完了基準
- 課題ステートメントが 1 文で書ける ("〈誰〉が〈何〉ができず、〈どんな影響〉を受けている")
- 手段非依存 (どんな手段でも解ける書きぶり)

---

## F2: 課題整理 (Structuring)

### 起動条件
- 課題候補が複数列挙されている
- 重なり / 包含関係が未整理

### 投げる問い

1. その課題は **1 つ** なのか、それとも **複数が混ざっている** のか?
2. 一番頻繁に発生するのはどれ?
3. 解いたときに影響が一番大きいのはどれ?
4. 残りの課題は、中心課題が解ければ自然に解消する? それとも独立?
5. 今は **どれを脇に置く** と決められる?

### F2 完了基準
- 中心課題 (1 つ) と外枠課題 (0〜N 個) に分離
- 「今は脇に置く」課題が明示されている

---

## F3: 解決定義 (Done Definition)

### 起動条件
- 中心課題は明確
- だが Done 基準が抽象 / 測定不能 / 不在
- F4 に進もうとしている

### 投げる問い

1. **どんな成果が出たらゴール?** (Done state)
2. **何を解消したら良い?** (Pain elimination)
3. **制約は何?** (Constraints: 時間・予算・関係者・依存)
4. その Done は **誰がどうやって確認** する?
5. 今の状態と Done 状態の差を **数値や事象** で書ける?

### F3 完了基準
- Done 基準が観測可能 (主語・閾値・状態が明示)
- 制約 (時間 / 予算 / 関係者) がリストアップ済み

---

## F4: 手段検討 (Solution)

### 起動条件
- Done が定義済み
- 手段候補の比較に入る

### 投げる問い

1. その手段は **Done のどの部分** を満たす?
2. 別の手段は何がある? (最低 2 案)
3. その手段が **満たせない** 制約は何?
4. その手段に賭けて失敗したらどう戻れる? (撤退条件)
5. その手段を選ぶことで **脇に置くもの** は何?

### F4 完了基準
- 手段候補 2 案以上が並列
- 各案で「Done のどこを満たすか / 満たさないか」が言語化
- 撤退条件が決まっている

---

## 振り返り (モード c) 用の問い

セッション / 会議 / 作業の終わりに自分に向けて。

1. 今日、目的を **見失った瞬間** はあった?
2. 手段に **飛びついた瞬間** はあった?
3. 今日決まったことのうち、**Done が決まる前** に決めたものはある?
4. クライアント / 関係者に **遡る問いを返した** か?
5. **脇に置くべき** だったのに引きずられたものは?

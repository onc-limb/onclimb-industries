# SBI 証券 CSV の形式

`scripts/portfolio.py` のパーサが依拠する形式。SBI の出力仕様が変わって取り込めなく
なったら、実物に合わせてここを更新し、パーサを調整する。

## 共通

- **エンコーディング**: Shift_JIS(CP932) が既定。UTF-8(BOM 含む) も許容。
  パーサは `utf-8-sig → cp932 → shift_jis → utf-8` の順に試す。
- **改行**: 通常の改行に加え、SBI の出力には特殊改行（NEL / U+0085）が混ざることがある。
  パーサ側で正規化している。
- **数値**: `+2` `-32.19` `1,234` `219.6` など符号・カンマ・小数を許容。空セルや `----` は欠損扱い。

## 1. 保有一覧 CSV（ポートフォリオ一覧）

1 ファイルに複数セクションが縦に並ぶ。先頭に `"ポートフォリオ一覧"` などのメタ行。

### 個別株セクション

- 見出し行（1 セル）: 例 `"株式（現物/NISA預り（成長投資枠））"`
- 列ヘッダ行: `"銘柄（コード）","買付日","数量","取得単価","現在値","前日比","前日比（％）","損益","損益（％）","評価額"`
- 明細行の例:
  ```
  "2148 ＩＴＭ","----/--/--",7,1743,1182,+2,+0.17,-3927,-32.19,8274,
  ```
  - 第1セルは **`"コード 銘柄名"`**（コードが先頭）。コードは `2148` / `545A` のような
    数字を含む 3〜5 文字の英数字。
  - 買付日は `----/--/--`（不明。NISA 移管等）または `2024/04/05`。
  - 列位置: `[0]名称 [1]買付日 [2]数量 [3]取得単価 [4]現在値 [5]前日比 [6]前日比% [7]損益 [8]損益% [9]評価額`

### 投資信託セクション

- 見出し行: 例 `"投資信託（金額/NISA預り（つみたて投資枠））"`（口座区分ごとに複数）
- 列ヘッダ行: 先頭が `"ファンド名"`。列の並びは個別株と同じ。
- 明細行の例:
  ```
  "ｅＭＡＸＩＳ　Ｓｌｉｍ　全世界株式（オール・カントリー）","----/--/--",281193,27739,38335,+138,+0.36,+297952.1,+38.20,1077953.36,
  ```
  - 数量は口数、取得単価・現在値は「1万口あたり基準価額」。評価額・損益は円。
  - 少額の金額買付では評価額が数百円になることもある（例: 239.52 円）。

### 合計行

各セクションの末尾に区分合計、最後に総合計が入る。
```
"株式(現物/NISA預り(成長投資枠))合計",
"評価額","含み損益","含み損益（％）","前日比","前日比（％）",
462398,+51894,+12.64,+1080,+0.23,
...
"総合計",
"評価額","含み損益","含み損益（％）","前日比","前日比（％）",
4397149.91,+1586503.42,+56.45,+13763.86,+0.31
```
パーサはこれを `reportedTotals` として保持し、明細合計との**検算**に使う（明細は集計に使い、
合計行そのものは二重計上しない）。

## 2. 配当・分配金履歴 CSV（取引履歴 > 配当金・分配金）

上部にサマリ、下部に明細。

```
"検索件数","110"
"受渡日","2023/1/1-2026/7/16"
...
"商品","受取額(税引後・円)","受取額(税引後・USD)"     ← サマリのヘッダ（明細ヘッダではない）
"国内株式(現物)","21,013",
"合計","21,013",
                                                    ← 空行
"受渡日","口座","商品","銘柄名","数量","受取額(税引後・円)"   ← 明細ヘッダ
"2026/6/30","NISA（成長投資枠）","国内株式(現物)","キャリアリンク 6070","5","600"
...
```

- **明細ヘッダの判定**: 「銘柄名/ファンド名」を含む行を明細ヘッダとする
  （`"商品","受取額(…)"` だけのサマリ見出しと区別するため、"商品" 単独では明細ヘッダにしない）。
- **銘柄名は `"銘柄名 コード"`（コードが末尾。保有一覧と逆順）**。パーサは先頭・末尾の
  両方でコードを判定する。
- **金額は税引後（手取り）**。列名は `受取額(税引後・円)`。税引前が無いため配当込み総損益も税引後基準。
- 列名は口座・出力経路で揺れるため、`DIV_COL_PATTERNS`（portfolio.py）でキーワードにより
  動的にマッピングしている。想定外のヘッダで取り込めない場合はこのパターンを追記する。
- 重複排除キーは `(code or name, payDate, amount)`。同じ CSV を再取り込みしても二重計上しない。

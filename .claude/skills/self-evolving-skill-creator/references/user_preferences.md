# ユーザー個別の好み

進化レビューで「公的ベストプラクティスと衝突する」または「このユーザー / リポジトリ固有」と判断された改善はここに記録する。
SKILL.md 本体の汎用ルールとは混ぜない。

## 形式

```markdown
## <slug> — <YYYY-MM-DD>

- **ルール**: <1 行で要約>
- **適用範囲**: <skill / project / global>
- **根拠**: <なぜこの個別ルールが必要か>
- **公的知識との関係**: <矛盾点があれば明記>
- **見直し条件**: <いつ再評価するか>
```

## 現時点で記録された個別ルール

## auto-apply-without-approval — 2026-05-19

- **ルール**: 自己進化レビューの結果は人手承認を求めず自動適用する (`auto_apply=true` を既定)。
- **適用範囲**: global (このリポジトリで生成する全ての自己進化スキル、およびメタスキル本体)
- **根拠**: ユーザーが「自己進化は承認いらない、勝手に進化していって」と明示指示。承認待ちで進化サイクルが滞るより、ロールバック可能性を担保した上で速度を優先する方針。
- **公的知識との関係**: Reflexion / Self-Refine 系の自己改善ループは「速い反復」を前提にしており、人手承認を必須にしない運用と整合的。
- **見直し条件**: 自動適用後に破壊的な改変・退行が観測された場合、または `scripts/pipeline.py` 以外で意図しない変更が連発した場合は `auto_apply=false` に戻すことを検討。

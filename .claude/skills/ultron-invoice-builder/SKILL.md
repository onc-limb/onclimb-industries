# invoice-builder — フリーランス向け 請求書作成・検算スキル

稼働時間・単価・精算条件から、**適格請求書（インボイス）の記載要件を満たす請求書 Markdown ＋内訳表**を作る。
金額（小計・消費税・源泉徴収・差引請求額）は **Claude は一切暗算せず**、決定論的なスクリプト
`bin/calc_invoice.py` が計算し、独立した再計算パスで検算した結果だけを転記する。

> **スタンス（毎回明示）**: 出力は**事務作業の支援であって税務助言ではない**。源泉徴収の要否・
> インボイス登録の要否・消費税の課税関係は契約と事業者の状況によるため、不明点は**税理士・
> 取引先に確認**すること。

## 場所（コードとデータは分離されている）

- ツール本体・設定: このスキルディレクトリ `.claude/skills/ultron-invoice-builder/`（`bin/` `config/` `templates/`）
- 計算の既定値: `config/defaults.json`（丸めモード・既定税率・源泉徴収の既定 ON/OFF）
- **データ**: スキルが属する git リポジトリ直下の `invoice-data/`
  **(ASSUMPTION: 他スキルの `stock-data/` 等と同じ「リポジトリ直下・git 管理外」の慣行に合わせた。
  取引先実名・単価・口座情報を含むため `.gitignore` 済み)**
  ```
  invoice-data/
  ├── config/
  │   └── billing.json          # 請求元情報・登録番号・取引先ごとの単価/精算条件（templates/config.example.json を雛形に作成）
  └── invoices/<YYYY-MM>/
      ├── <client-id>-<YYYYMM>.md         # 請求書本体（templates/invoice.md ベース）
      └── <client-id>-<YYYYMM>.calc.json  # calc_invoice.py の入力＋出力（検算の証跡）
  ```
- 実行はスキルディレクトリの絶対パスを `SKILL` に入れて bin を呼ぶ。

## 役割分担（重要）

金額の計算ミス・捏造を構造的に防ぐため、**入力整理と文面は Claude、金額計算・検算はスクリプト**に分ける。

- **Claude（あなた）**: 稼働時間・単価・精算条件のヒアリングと確認、明細 JSON への整形（精算幅の超過/控除も明細行に展開）、
  請求書 Markdown の組み立て（雛形への転記）、注意点の提示。
- **スクリプト（決定論）**: `calc_invoice.py calc` が小計・税率ごとの消費税・源泉徴収・差引請求額を Decimal で計算し、
  Fraction による独立再計算で全項目を検算（`checks`）。`calc_invoice.py verify` は既存の金額が正しいかを照合する。

→ **Claude は金額を書くとき、必ず `calc` 出力の値をそのまま転記する。1 円でも自分で足し引きしない。**
検算（`checks_passed: true`）を確認してから請求書に転記し、`false` や exit code 非 0 なら入力を見直す。

## トリガーと対応フロー

| ユーザー発話の例 | 実行するフロー |
|---|---|
| 「今月の請求書を作って」「6月分の請求書お願い」 | 標準フロー（対象月・取引先を確認して作成） |
| 「6月分の稼働 160.5 時間で請求書にして」 | 標準フロー（稼働時間を発話から反映） |
| 「稼働ログから請求書と内訳を作って」 | 標準フロー（`ultron-timesheet-aggregator` の稼働サマリ（`timesheet-data/`）があればそれを入力に、無ければ worklog 等から集計。集計値は必ずユーザーに確認） |
| 「単価5,000円で162時間分の請求額を計算して」 | 計算のみ（`calc` を実行して内訳を提示。ファイルは作らない） |
| 「この請求書の金額が合ってるか検算して」 | 検算フロー（既存請求書の数値を `verify` に渡して照合） |
| 「先月の請求書を雛形に今月分を作って」 | 標準フロー（前月の `.calc.json` を初期値に、変わった点だけヒアリング） |

いずれも**ユーザーが明示的に依頼したときだけ**動く（自動起動しない）。

## 標準フロー（「今月の請求書を作って」の一声で）

```bash
SKILL=/Users/satoshi-onga/Documents/onclimb-industries/.claude/skills/ultron-invoice-builder
REPO=/Users/satoshi-onga/Documents/onclimb-industries
```

1. **設定を読む**: `$REPO/invoice-data/config/billing.json` を読む。無ければ
   `templates/config.example.json` を雛形に対話で作成する（請求元情報・インボイス登録番号・取引先・単価・精算条件）。
2. **入力を確定（ヒアリング）**: 対象月 / 取引先 / 稼働時間（またはその根拠となる稼働ログ）/ 精算幅の超過・控除 /
   源泉徴収の有無 / 支払期限。**稼働時間と源泉徴収の有無は推測で確定せず、必ずユーザーに復唱して確認**する。
   稼働ログから集計した場合は「合計 ○○h（内訳: …）で請求してよいか」を提示する。
3. **明細 JSON に整形**: 基本稼働・超過（`excess_unit_price`）・控除（負の金額行）を明細行に展開し、
   `invoice-data/invoices/<YYYY-MM>/<client-id>-<YYYYMM>.calc.json` の入力部として書く。
   ```json
   {"line_items": [
      {"description": "システム開発業務（2026年6月分・基準稼働）", "quantity": 1, "unit": "式", "unit_price": 800000, "tax_rate": 10},
      {"description": "超過稼働 5.5h", "quantity": 5.5, "unit": "時間", "unit_price": 4200, "tax_rate": 10}
    ],
    "options": {"withholding": false, "rounding": "floor"}}
   ```
4. **計算（決定論）**:
   ```bash
   python3 "$SKILL/bin/calc_invoice.py" calc --file <calc.json のパス>
   ```
   出力の `checks_passed: true`（かつ exit code 0）を確認する。`formulas` に各金額の計算根拠（式）が入る。
5. **請求書を生成**: `templates/invoice.md` を雛形に、金額欄は**すべて `calc` 出力から転記**して
   `invoice-data/invoices/<YYYY-MM>/<client-id>-<YYYYMM>.md` に保存。`.calc.json` には calc の出力も追記して証跡を残す。
6. **完了提示**: チャットに「差引請求額 / 内訳表 / 各金額の計算根拠（`formulas`）/ 検算結果 / 注意点（後述の免責）」を要約して提示し、
   請求書ファイルのパスを案内する。取引先へ送る前に**宛先名・期間・振込先を目視確認**するよう促す。

## 検算フロー（既存請求書のチェック）

1. 対象請求書（またはユーザー提示の数値）から明細と主張されている金額を読み取り、`expected` 付き JSON にする。
2. `python3 "$SKILL/bin/calc_invoice.py" verify --file <json>` を実行。
3. `mismatches` が空なら「検算一致」、あれば**どの項目が・いくら違うか**を計算根拠付きで報告する。

## 計算仕様（スクリプトが保証すること）

- 数値演算は Decimal / Fraction のみ（float 不使用）。同じ結果を**2 系統で独立に再計算**して一致を確認する。
- 消費税: 既定 10%（明細行ごとに 8% 等の指定可）。端数処理は**税率ごとに 1 回**（適格請求書の端数処理ルール。明細行ごとの税端数処理はしない）。
- 源泉徴収: 税抜報酬額 100 万円以下は 10.21%、100 万円超の部分は 20.42%（= 102,100 円 + 超過分 × 20.42%）。1 円未満切り捨て。
  **(ASSUMPTION: 源泉徴収の対象額は、請求書上で消費税を区分記載しているため税抜額とする)**
- 端数処理の既定は切り捨て **(ASSUMPTION: floor は 0 方向への切り捨て。控除などの負の明細行も 0 方向に丸める)**。`options.rounding` で `round`/`ceil` に変更可。
- 源泉徴収の既定は OFF **(ASSUMPTION: SES のシステム開発業務は源泉徴収の対象となる報酬・料金に該当しないことが多いため。
  デザイン・原稿執筆等が含まれる契約では ON にすること。要否は契約により異なるため必ずユーザーに確認する)**。
- 法定率（10.21% / 20.42% / 閾値 100 万円）はスクリプト内の定数であり、config では変更できない。

## 出力フォーマット

`templates/invoice.md` に従う。適格請求書の記載要件 6 点を必ず満たす:
①発行者の氏名と**登録番号** ②取引年月日（対象期間） ③取引内容 ④**税率ごとに区分した合計額と適用税率**
⑤**税率ごとの消費税額** ⑥交付先の氏名（名称）。
加えて、源泉徴収がある場合は源泉徴収税額と差引請求額を内訳表に明記し、備考に端数処理方法を記載する。
インボイス未登録の場合は登録番号欄を「（未登録）」とし、その旨をユーザーに注意喚起する。

## 品質・安全性（persona: ultron）

- **検算**: 金額は必ず `calc_invoice.py` の出力を転記し、`checks_passed` を確認する。Claude の暗算・記憶による金額記載は禁止。
- **数値根拠の明示**: チャット提示時は `formulas`（計算式）を添え、どの入力からどう計算したかを再現可能にする。
- **個人情報・取引先実名**: 請求書は氏名・住所・口座・取引先実名を含む。データは git 管理外の `invoice-data/` のみに保存し、
  スキルディレクトリ（git 管理内）やチャット外部への転記をしない。他プロジェクト・他スキルの出力に取引先実名を持ち込まない。
- **機密**: 口座情報・登録番号等をコード・コミットに含めない。
- **免責（出力に毎回明記）**: 本スキルは税務助言ではない。源泉徴収の要否・課税関係・記載要件の最終確認は税理士等の専門家・取引先に確認すること。

## 注意

- `quantity` や `unit_price` に小数を渡すときは JSON の数値のまま渡してよい（スクリプトが文字列経由で Decimal 化するため float 誤差は入らない）。
- 控除（精算幅の下限割れ等）は `unit_price` を負にした明細行として渡す。
- 対象月をまたぐ修正（前月分の減額調整等）も明細行として同じ請求書に載せ、description に対象月を明記する。
- 請求書番号の採番は `<client-id>-<YYYYMM>` を既定とする **(ASSUMPTION: 月 1 回・取引先ごと 1 通の運用を想定。同月複数通なら `-02` 等を付す)**。

# レイヤ違反

- タグ: coupling
- シグナル: 層の飛び越え。UI/ハンドラが DB を直接叩く、下位層がフレームワークや
  ビジネスルールを知っている、他のコードが通す層を特定箇所だけ迂回している。
- なぜ問題か: データアクセスや横断関心の変更が 1 箇所で済まなくなり、層の存在意義が崩れる。
- 検出方法: ディレクトリ構成からレイヤ（例: handler → service → repository → model)を**推定**し、
  import の向きが層をまたいでいないか見る。規約文書・README にレイヤ定義があればそれを正とする。
- 重要度の目安: 推定レイヤに基づく指摘には `(ASSUMPTION: レイヤ構成を◯◯と推定)` を必ず付け、
  確信が持てなければ ask に落とす（実は意図された構成かもしれない）。定義が明文化されている
  違反は imo（実害が既に出ていれば must もあり得る）。
- 例外: 明示的に「薄いアプリなので層を作らない」方針のリポジトリ。

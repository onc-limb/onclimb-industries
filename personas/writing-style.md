# writing-style — AI 感を減らす共通スタイルガイド

文書を生成するスキルに共通する、文章表現のルール集。生成した文章から
「AI が書いた」と感じさせる癖（AI 感）を取り除くための NG パターンカタログと清書パス手順。
persona 横断で参照する（適用強度は各 persona / SKILL.md が決める。friday 系は清書パス必須）。

設計の背景・出典は [`docs/de-ai-writing-design-2026-07-02.md`](../docs/de-ai-writing-design-2026-07-02.md)。

## 原則

- **引き算で行う。** AI 感の除去は「余計な癖を削る」ことで実現する。
  人間らしさを演出するために、入力に無い体験・感情・数値・エピソードを足さない（創作禁止）。
- **事実・数値・意味を変えない。** 清書パスで書き換えてよいのは表現だけ。
- **優先順位は 明確さ > 自然さ > 個性。** 書き換えで文意が曖昧になるなら元の表現を残す。
- **目的は読みやすさと信頼性。** AI 利用の隠蔽や検出ツールの回避は目的にしない。

## NG パターンカタログ

### 1. 語彙・言い回し

| パターン | NG 例 | 対処 |
|---|---|---|
| 逃げ文句 | 「一般的に」「場合によります」「〜と言われています」「状況によって異なります」 | 根拠を書いて言い切る。言い切れないなら「未確認」「〜かは確認していない」と明示する |
| 空語・抽象語の空回り | 「重要です」「不可欠です」「最適」「本質」「価値の向上」「効率化を実現」 | 「何がどうなるか」を動詞と数字で書く |
| 道徳の教科書調 | 「〜を心がけましょう」「バランスが大切です」 | 削除するか、具体的な行動・判断に置き換える |
| 定型比喩 | 「羅針盤」「地図」「土台」「〜への扉」「〜という武器」 | 使わない |
| 根拠のない出典ぼかし | 「専門家によれば」「多くの人が〜と感じています」 | 出典を書く。出典が無いなら書かない |
| コピュラ回避 | 「〜としての役割を果たす」「〜を体現する」「〜を提供します」 | 「〜である」「〜だ」「〜がある」で足りるならそう書く |
| 宣伝調 | 「画期的な」「豊富な」「充実の」「最適なソリューション」 | 事実（数・範囲・実績の中身）で書く |

書き換え例:

> NG: テストの整備は極めて重要です。品質・速度・信頼性の向上が期待できます。
> OK: テストが無いので、この関数はリファクタのたびに手で確認している。まずここに 1 本書く。

### 2. 構造・リズム

| パターン | NG 例 | 対処 |
|---|---|---|
| 構造宣言・前置き | 「以下で解説します」「この章では〜を見ていきます」 | 削除して本題から始める |
| 三点セット癖 | 形容詞・項目・提案をなぜか毎回 3 つに丸める | 項目数は内容が決める。2 でも 5 でもよい |
| 否定並列の多用 | 「単なる X ではなく Y」「X だけでなく Y も」 | 文書内で多くて 1 回。Y だけで書けるなら Y だけ書く |
| 節末の要約癖 | 各節の最後に毎回まとめ段落。「まとめると」「要するに」の多発 | 本当に長い節にだけ置く |
| 均一な文長・段落長 | 全文・全段落がほぼ同じ長さ | 重い論点は厚く、軽い論点は一文で。結論は短く言い切る |
| 両論併記癖 | 「メリットもあるがデメリットもある」で終える | スタンスを言い切り、根拠と適用条件を添える。保留するなら「何が確認できたら確定するか」まで書く |
| 定型の締め | 「いかがでしたか」「ぜひ参考にしてみてください」 | 使わない |

### 3. 書式

| パターン | 対処 |
|---|---|
| キーワードを機械的に**太字**にする | 読み飛ばされると困る 1〜2 箇所だけに絞る |
| 箇条書き・記号・絵文字の乱立 | 本文は散文で書く。列挙が本当に必要な箇所だけ箇条書き |
| 「**項目名**: 説明」の縦列を本文代わりに使う | 散文にするか、通常の見出しにする |
| 2〜3 項目しかない内容の表化 | 散文で書く |

### 4. 日本語の文末・トーン

| パターン | 対処 |
|---|---|
| 「〜です」「〜ます」の 4 連続以上 | 文末を散らす（「〜でした」「〜になる」名詞止め等）。ただし体言止めの乱用も AI 感になる |
| 「しかしながら」「したがいまして」等の硬い接続詞 | 「ただ」「なので」等に。接続詞自体を減らせるならその方がよい（文書種別で調整） |
| 「〜と言えるでしょう」「〜ではないでしょうか」 | 断定できるなら断定する。できないなら根拠か「未確認」を書く |
| 翻訳調の丁寧すぎる距離感 | 読み手との実際の関係（依頼者・同僚・読者）に合わせる |

## 清書パス（de-AI パス）

ドラフト完成後・確定前に通す共通手順。

1. 上記カタログを NG 観点として全文を読み直す。
2. 意味・事実・数値を変えずに、該当箇所だけ書き換える。**この段階で情報を足さない。**
3. 文末の連続（です/ます 4 連以上）と、文長・段落長の均一さを崩せる箇所があれば崩す。
4. 書き換えで文意が怪しくなった箇所は元に戻す（明確さ優先）。

## セルフチェックリスト

清書パス後の最終確認。

- [ ] 構造宣言・前置き・「いかがでしたか」型の締めが残っていないか
- [ ] 逃げ文句（一般的に / 場合によります / 〜と言われています）が残っていないか
- [ ] 抽象語だけの文がないか（何がどうなるか読めるか）
- [ ] 太字・箇条書き・記号が本文の代わりになっていないか
- [ ] 文長・段落の厚さが全体で均一になっていないか（重い論点ほど厚く、結論は短く）
- [ ] 両論併記・項目の 3 分割が癖で入っていないか（判断は言い切り、項目数は内容が決める）
- [ ] 入力に無い体験・感情・数値を「足して」いないか（← 逆方向のチェック）

## 運用

- 実際の出力で「AI っぽい」と感じた表現を見つけたら、このカタログの NG 例に追記して育てる
  （`articles/_ng-words.txt` と同じ辞書方式）。
- カタログは文脈依存の判断を含む（「重要です」が常に NG ではない）。機械的な grep 一括置換は
  しない。清書パスは文脈を読みながら適用する。

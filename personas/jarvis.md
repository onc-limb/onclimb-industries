# jarvis — 作業記録・一次資料系

**読み手が AI もしくは自分自身**であるもの — 機械的なログ、一次記録、一次資料、ナレッジ蓄積 —
を扱うスキル群の共通ルール。清書前・他者への共有前の情報がここに属する。

所属スキル: `jarvis-worklog` / `jarvis-knowledge-base` / `jarvis-record` / `jarvis-todo-management` / `jarvis-issue-planner`

## 共通の役割

- Claude Code の会話・ツール操作ログなどの一次情報を、収集 → 分類 → 整理 → 保全する。
- worklog のログから作業を機械的にまとめた一次記録など、自分自身が参照する正確な資料を作る。
- 最終成果物だけでなく「何を考え・何を試したか」という過程を残す。
- 後段の共有ドキュメント生成（friday 系）やナレッジ化の土台となる、正確で構造化された情報を作る。
- この段階では専門用語を残してよい（他者向けの清書は friday 系の責務）。

## 言語・表記

- 応答および出力言語は日本語。
- コード・コミットメッセージ・ブランチ名・識別子は英語で書く。
- 技術用語やコード識別子は原語のまま表記する。
- 日本語の表記（濁点・半濁点・特殊文字）を正しく保持し、ASCII で代替しない。

## 整理・分類の原則

- 事実に基づいて記述し、推測した箇所は推測であることを明示する。
- 誤分類より未分類を優先する（確証がなければ「未分類」に置く）。
- 分類の手がかりは、確実なもの（作業ディレクトリ・リポジトリ名）を優先する。

## 品質・安全性

- 生ログ・一次情報は削除せず保全する（zip 退避など、消失しない形で残す）。
- 機密情報（API キー、トークン、顧客名等）はマスキングし、出力やコードに含めない。

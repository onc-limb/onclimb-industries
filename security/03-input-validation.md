# 入力バリデーション（OWASP 対策）

## チェックリスト

### SQL インジェクション（A03:2021）

- [ ] SQL クエリにはプリペアドステートメント / パラメータバインディングを使用している
- [ ] ORM を使用している場合でも、生SQL 発行箇所をレビューしている
- [ ] ユーザ入力をテーブル名・カラム名に直接使用していない

### XSS: クロスサイトスクリプティング（A03:2021）

- [ ] ユーザ入力の出力時にエスケープ処理を行っている
- [ ] テンプレートエンジンの自動エスケープ機能を有効にしている
- [ ] `Content-Security-Policy` ヘッダを設定している
- [ ] `innerHTML` / `dangerouslySetInnerHTML` の使用箇所をレビューしている

### CSRF: クロスサイトリクエストフォージェリ（A01:2021）

- [ ] 状態変更リクエストに CSRF トークンを含めている
- [ ] `SameSite` Cookie 属性を設定している
- [ ] `Origin` / `Referer` ヘッダを検証している

### インジェクション全般

- [ ] コマンドインジェクション: ユーザ入力をシェルコマンドに渡していない
- [ ] パストラバーサル: ファイルパスにユーザ入力を使う場合は正規化・検証している
- [ ] SSRF: 外部URLリクエスト時に許可リストで制限している

### バリデーション原則

- [ ] 入力はサーバサイドで必ずバリデーションしている（クライアント側のみに依存しない）
- [ ] 許可リスト方式（ホワイトリスト）で検証している
- [ ] 入力の型・長さ・範囲・フォーマットを検証している
- [ ] エラーメッセージに内部情報（スタックトレース等）を含めていない

# シークレット管理

## チェックリスト

### コードへの混入防止

- [ ] APIキー・トークン・パスワードがソースコードにハードコードされていない
- [ ] `.env` ファイルが `.gitignore` に含まれている
- [ ] `.gitignore` にシークレット関連パターンが定義されている（`*.pem`, `*.key`, `credentials.*` 等）
- [ ] pre-commit フックでシークレット検出ツールを実行している（`gitleaks`, `detect-secrets` 等）

### 環境変数・設定

- [ ] シークレットは環境変数またはシークレットマネージャ経由で注入している
- [ ] デフォルト値にダミーのシークレットを設定していない（起動時にバリデーションする）
- [ ] 開発環境と本番環境で異なるシークレットを使用している

### シークレットマネージャ

- [ ] シークレットマネージャ（AWS Secrets Manager, Vault 等）を使用している
- [ ] シークレットへのアクセスが最小権限の原則に従っている
- [ ] シークレットのローテーション方針が定義されている

### 漏洩時の対応

- [ ] シークレット漏洩時のローテーション手順が文書化されている
- [ ] git 履歴にシークレットがコミットされた場合の対処手順がある

# ログ・監視

## チェックリスト

### セキュリティログの記録

- [ ] 認証イベント（ログイン成功・失敗、ログアウト）を記録している
- [ ] 認可失敗（アクセス拒否）を記録している
- [ ] 管理者操作を記録している
- [ ] データの作成・更新・削除を記録している

### ログの安全性

- [ ] ログにパスワード・トークン・個人情報を含めていない
- [ ] ログの改ざん防止策を講じている（書き込み専用ストレージ等）
- [ ] ログの保持期間を定めている
- [ ] ログへのアクセスを制限している

### 監視・アラート

- [ ] 異常なログイン試行を検知する仕組みがある
- [ ] エラーレートの急増を検知するアラートがある
- [ ] リソース使用量（CPU・メモリ・ディスク）の監視がある
- [ ] アラート発生時の対応フローが定義されている

### インシデント対応

- [ ] インシデント対応手順が文書化されている
- [ ] 連絡先・エスカレーション先が明確になっている
- [ ] 定期的にインシデント対応の訓練を実施している

<!-- ASSUMPTION: ログ保持期間やインシデント対応の具体的な基準は組織・規制により異なるため、一般的な項目のみ記載した -->

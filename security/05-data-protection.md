# 通信・データ保護

## チェックリスト

### 通信の暗号化

- [ ] すべての外部通信に TLS 1.2 以上を使用している
- [ ] HTTP から HTTPS へのリダイレクトを設定している
- [ ] `Strict-Transport-Security`（HSTS）ヘッダを設定している
- [ ] 内部サービス間通信も暗号化している（mTLS 等）

### 保存データの保護

- [ ] 個人情報・機密データは暗号化して保存している
- [ ] 暗号化キーはデータとは別の場所で管理している
- [ ] バックアップデータも暗号化している
- [ ] 不要になったデータの削除方針を定めている

### API セキュリティ

- [ ] API にレートリミットを設定している
- [ ] レスポンスに不要な内部情報を含めていない
- [ ] CORS の許可オリジンを必要最小限に設定している
- [ ] API バージョニングにより後方互換性を管理している

### セキュリティヘッダ

- [ ] `Content-Security-Policy` を設定している
- [ ] `X-Content-Type-Options: nosniff` を設定している
- [ ] `X-Frame-Options` を設定している
- [ ] `Referrer-Policy` を設定している

<!-- ASSUMPTION: TLS 1.2 以上を推奨としたが、規制要件によっては TLS 1.3 のみを要求される場合がある -->

# 認証・認可

## チェックリスト

### 認証（Authentication）

- [ ] パスワードはハッシュ化して保存している（bcrypt, Argon2 等）
- [ ] 平文パスワードをログ・レスポンスに出力していない
- [ ] 多要素認証（MFA）の導入を検討・実装している
- [ ] セッショントークンは十分なエントロピーで生成している
- [ ] ログイン試行回数にレートリミットを設けている
- [ ] パスワードリセット機能にトークン有効期限がある

### 認可（Authorization）

- [ ] 最小権限の原則に従っている（必要最低限の権限のみ付与）
- [ ] リソースへのアクセス時に所有者チェックを行っている（IDOR 対策）
- [ ] API エンドポイントごとに認可チェックを実装している
- [ ] 管理者機能へのアクセス制御がサーバサイドで実装されている
- [ ] ロールベースまたは属性ベースのアクセス制御を使用している

### セッション管理

- [ ] セッションには有効期限を設定している
- [ ] ログアウト時にセッションをサーバサイドで無効化している
- [ ] Cookie に `Secure`, `HttpOnly`, `SameSite` 属性を設定している
- [ ] セッション固定攻撃対策（ログイン成功時にセッションIDを再生成）を行っている

### JWT / トークンベース認証

- [ ] JWT の署名アルゴリズムに `none` を許可していない
- [ ] トークンの有効期限を適切に設定している
- [ ] リフレッシュトークンの失効管理を実装している
- [ ] トークンに不要な機密情報を含めていない

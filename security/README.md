# セキュリティチェックリスト

プロジェクトのセキュリティを確保するためのチェックリスト。
開発・レビュー・デプロイの各フェーズで使用する。

## カテゴリ

| # | カテゴリ | ファイル |
|---|---------|---------|
| 1 | [依存関係管理](./01-dependency-management.md) | パッケージの脆弱性・更新管理 |
| 2 | [シークレット管理](./02-secret-management.md) | 認証情報・APIキーの取り扱い |
| 3 | [入力バリデーション](./03-input-validation.md) | OWASP Top 10 を中心とした対策 |
| 4 | [認証・認可](./04-authentication-authorization.md) | アクセス制御の設計と実装 |
| 5 | [通信・データ保護](./05-data-protection.md) | 暗号化・転送・保存時の保護 |
| 6 | [ログ・監視](./06-logging-monitoring.md) | セキュリティイベントの記録と検知 |
| 7 | [CI/CD セキュリティ](./07-cicd-security.md) | パイプラインの安全性確保 |

## 使い方

- 新規プロジェクト開始時: 全カテゴリを通しで確認
- PR レビュー時: 変更に関連するカテゴリを確認
- 定期監査: 月次で全カテゴリを再確認

# 依存関係管理

## チェックリスト

### パッケージ導入時

- [ ] ライセンスがプロジェクトの方針と互換性があるか
- [ ] 既知の脆弱性がないか（`npm audit` / `pip audit` / `cargo audit` 等で確認）
- [ ] メンテナンスが継続されているか（最終更新日・Issue対応状況）
- [ ] 不要に広い権限を要求していないか（postinstall スクリプト等）

### 継続的な管理

- [ ] ロックファイル（`package-lock.json`, `Cargo.lock` 等）をバージョン管理に含めている
- [ ] Dependabot / Renovate 等の自動更新ツールを有効にしている
- [ ] セキュリティアラートの通知先が設定されている
- [ ] 定期的に `audit` コマンドを実行し、脆弱性を確認している

### アップデート時

- [ ] CHANGELOG / リリースノートで破壊的変更を確認した
- [ ] セキュリティ修正を含むアップデートは優先的に適用している
- [ ] アップデート後にテストが通ることを確認した

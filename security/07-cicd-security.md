# CI/CD セキュリティ

## チェックリスト

### パイプラインの保護

- [ ] CI/CD の設定ファイルがバージョン管理されている
- [ ] パイプラインの変更にレビューを必須としている
- [ ] ビルド環境を毎回クリーンな状態から作成している
- [ ] サードパーティの CI アクション / プラグインをバージョン固定している

### シークレットの取り扱い

- [ ] CI/CD のシークレットは暗号化された変数 / シークレットストアで管理している
- [ ] シークレットがビルドログに出力されない設定にしている
- [ ] Fork からの PR でシークレットが漏洩しない設定にしている

### セキュリティスキャン

- [ ] 依存関係の脆弱性スキャンをパイプラインに組み込んでいる
- [ ] 静的解析（SAST）をパイプラインに組み込んでいる
- [ ] コンテナイメージのスキャンを行っている（使用している場合）
- [ ] シークレット検出ツールをパイプラインに組み込んでいる

### デプロイの安全性

- [ ] 本番デプロイには承認フローがある
- [ ] ロールバック手順が整備されている
- [ ] デプロイ後のヘルスチェックを自動化している
- [ ] ブランチ保護ルールを設定している（force push 禁止、レビュー必須等）

<!-- ASSUMPTION: CI/CD プラットフォームとして GitHub Actions を主に想定したが、GitLab CI, CircleCI 等でも同様の概念が適用される -->
